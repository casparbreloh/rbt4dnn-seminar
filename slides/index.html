01 · ProblemWhy requirements?

Accuracy answers how often.
Requirements ask when.

A model can perform well on a test set while violating a necessary condition in a rare, safety-relevant scenario.

Conventional testingTest set → aggregate metric

Summarizes behavior over the sampled distribution.

Requirements-based testingScenario → expected behavior

Tests a stated input condition against its postcondition.

[1] Mozumder et al., 2025, §1–202 / 16
Research gapPixels ↔ requirements

Formal test generation needs a specification. DNN requirements resist one.

Requirements use semantic concepts—vehicles, lanes, distance—while learned components and generators operate on pixels.

GAP

Formalization

Precise input predicates are difficult to write for visual domains.

IDEA

Semantic space

Use glossary terms to express preconditions.

CLAIM

Targeted tests

Fine-tune a generator toward each requirement.

RQ6What do failures reveal?

ImageNet failures exposed repeatable decision biases.

Background objects and partial animal views drove systematic misclassifications, revealing model-specific patterns and single-label dataset limitations.

Threats to validityThe semantic measurement loop

The central uncertainty is whether the generated scene means what the test says.

LABEL

What is present?

Scene graphs, morphology, human labels, or VQA define glossary terms.

MEASURE

Does it match?

Glossary-term classifiers operationalize consistency and diversity.

REVIEW

Is it credible?

Selected failures receive multi-assessor human review.

Hard case: precise relations such as “within 10 meters” remained difficult for generation and assessment.

Independent critiqueEvidence boundary

A detected failure needs two labels.

First: the generated image satisfies the precondition. Second: the model violates the postcondition.

VALID ∧ FAILthe joint outcome that constitutes defensible requirement-failure evidence

CritiqueSemantic labor

The hard part is still human.

Validity depends on four linked decisions: what the requirement means, how visual terms are defined, how labels are assigned, and whether evaluators agree.
